### **1Ô∏è‚É£ Full Syntax**

```sql
CREATE [OR REPLACE] [FORCE | NOFORCE] VIEW view_name
    [(alias1, alias2, ...)]
AS
    subquery
    [WITH CHECK OPTION [CONSTRAINT constraint_name]]
    [WITH READ ONLY];

-- Drop a view
DROP VIEW view_name;
```

---

### **2Ô∏è‚É£ Example ‚Äì Create Views**

#### **a) Simple View**

```sql
CREATE VIEW vw_student_names AS
SELECT student_id, first_name, last_name
FROM students;
```

#### **b) View with Column Aliases**

```sql
CREATE VIEW vw_course_info (CourseID, CourseName, Fees) AS
SELECT course_id, course_name, course_fee
FROM courses;
```

#### **c) Complex View (Join)**

```sql
CREATE VIEW vw_student_course AS
SELECT s.student_id,
       s.first_name || ' ' || s.last_name AS full_name,
       c.course_name,
       c.course_fee
FROM students s
JOIN courses c
     ON s.course_id = c.course_id;
```

#### **d) View with `WITH CHECK OPTION`**

```sql
CREATE OR REPLACE VIEW vw_active_students AS
SELECT student_id, first_name, last_name, status
FROM students
WHERE status = 'ACTIVE'
WITH CHECK OPTION;
```

*(Prevents inserting/updating rows through the view that don‚Äôt match the filter.)*

#### **e) Read-Only View**

```sql
CREATE OR REPLACE VIEW vw_all_courses AS
SELECT *
FROM courses
WITH READ ONLY;
```

---

### **3Ô∏è‚É£ Explanation**

* **`VIEW`** ‚Üí a saved query that behaves like a virtual table.
* **`OR REPLACE`** ‚Üí allows modifying the view without dropping it first.
* **`FORCE`** ‚Üí creates the view even if the base table doesn‚Äôt exist yet.
* **`WITH CHECK OPTION`** ‚Üí enforces the `WHERE` condition for DML operations via the view.
* **`WITH READ ONLY`** ‚Üí prevents any DML via the view.
* Views **do not store data** (by default); they store the query.
